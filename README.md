# VideoAutoCategorizer

一个基于 `tkinter` 的本地桌面工具，提供两大功能：

1. 重复视频分析与清理（可手动选择保留文件）
2. 批量转码（支持断点标记、停止后续跑、参数推荐）

主程序文件：`categories.py`

## 环境要求

- Python 3.10+
- 已安装并可在命令行访问：
  - `ffmpeg`
  - `ffprobe`

## 运行方式

```powershell
python categories.py
```

## 功能概览

### 1) 重复视频分析

- 支持添加多个目录扫描视频文件
- 匹配方式：
  - 文件名规则匹配（严格提取番号）
  - 可选 MD5 精确匹配（默认关闭）
- 可选元数据分析：
  - 分辨率 / 码率 / 时长（默认关闭）
- 0 字节视频会自动过滤
- 匹配结果支持分页显示（避免超长列表渲染异常）
- 每组可勾选：
  - `不去除重复视频`（该组整体跳过删除）
- `推荐选中保留`（只对当前页生效）：
  - 先按是否达到 720p 分层
  - 同层内按优先级：格式 > 容量(小) > 分辨率(高) > 码率(高)
- 删除前弹出高风险确认，并可滚动查看本次删除清单与原因
- 删除后仅移除已处理组，未处理组与勾选状态保留

### 2) 批量转码

- 支持分辨率、编码器、CRF/CBR 参数
- 分辨率自动推荐参数（例如 1080p / 720p）
- 支持编码器（含 GPU）：
  - `libx264` / `libx265` / `libsvtav1`
  - `h264_nvenc` / `hevc_nvenc` / `av1_nvenc`
  - `h264_amf` / `hevc_amf` / `av1_amf`
- 小于指定大小可跳过转码（1~5GB，默认 2GB）
- 可按目标参数跳过“已低于目标”的视频（主要在 CBR 下生效）
- 支持停止转码：
  - 当前文件完成后停止
  - 在临时目录写入断点标记（`目录名_下一个文件名`）
  - 下次可自动续跑
- 显示两级进度：
  - 总进度（含 ETA 预计剩余时间）
  - 当前文件进度
- 转码后文件名自动加 `_Hellman`，下次扫描识别后自动跳过
- 每个文件完成后会清理对应临时文件
- 转码后保留原文件时间信息（修改/创建时间对齐原文件）

## 使用建议

1. 先在“重复视频分析”中确认匹配结果再执行删除。
2. 初次使用转码建议先小批量测试参数和编码器兼容性。
3. 若使用 GPU 编码，建议同时观察显卡驱动状态与 ffmpeg 编码支持。

## 注意事项

- 删除操作不可恢复，请先确认保留项。
- 不同显卡/ffmpeg 版本对部分编码器参数支持可能不同。
- 该工具为本地文件处理工具，请确保目录权限可读写。
